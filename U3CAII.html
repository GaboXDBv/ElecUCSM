¿Qué es Sugeno?

El método Sugeno, también llamado Takagi-Sugeno-Kang (TSK), es un tipo de inferencia difusa donde las reglas mantienen la estructura SI ... ENTONCES ..., pero el consecuente ya no es un conjunto difuso como en Mamdani, sino una expresión matemática, una constante o una función lineal. Eso hace que el sistema sea más práctico cuando se busca una salida numérica directa, una implementación computacional más eficiente y una integración sencilla con control y modelado.

1. Idea central

En Sugeno, la parte difusa está principalmente en el antecedente de las reglas. Por ejemplo: SI error es pequeño Y cambio del error es negativo, ENTONCES salida = 0.4 o salida = 2x + y + 1. Eso significa que el sistema usa conjuntos difusos para interpretar las entradas, pero produce una salida numérica calculada a partir de cada regla activada.

2. ¿Por qué se usa?

Se usa mucho porque es más eficiente cuando el sistema debe calcular rápido, cuando se requiere una salida continua, o cuando se necesita combinar lógica difusa con ecuaciones matemáticas. En control automático y modelado, esto ayuda a obtener respuestas suaves y fáciles de implementar en software.

3. Estructura de una regla

La forma general de una regla de Sugeno es: SI x es A Y y es B ENTONCES z = f(x,y). En esa expresión, A y B son conjuntos difusos, mientras que f(x,y) puede ser una constante o una función lineal. La activación de la regla depende del grado de pertenencia de las entradas y la salida final se obtiene con una ponderación de todas las reglas activadas.

4. Rasgo distintivo

Lo que más diferencia a Sugeno es que no necesita una defusificación clásica como el centroide de Mamdani. En vez de eso, cada regla produce un valor numérico y luego esos valores se combinan usando un promedio ponderado. Por eso suele decirse que Sugeno es más directo en la salida.

Resumen rápido: Sugeno es un sistema difuso donde las entradas se interpretan con conjuntos difusos, pero la salida de cada regla es una constante o una ecuación. Luego se hace una ponderación entre las reglas activadas para obtener el valor final.

Tipos de Sugeno

Normalmente, al estudiar Sugeno se habla sobre dos formas principales según el tipo de consecuente que utiliza la regla.

Sugeno de orden cero

En este caso, cada regla tiene como salida una constante. Por ejemplo: SI temperatura es alta ENTONCES salida = 0.8. Es el tipo más simple de Sugeno y es útil cuando quieres que cada situación produzca directamente un valor fijo.

En MATLAB, esto se reconoce porque en la variable de salida las funciones de membresía del output pueden configurarse como constant. Esta es justamente una de las opciones que se observa en el editor de membership functions.

Sugeno de primer orden

En este caso, el consecuente es una función lineal de las entradas. Por ejemplo: SI error es positivo ENTONCES salida = 0.5x + 0.2y + 1. Este tipo es más flexible porque la salida se ajusta según el valor de las entradas, en lugar de dar siempre un número fijo.

En MATLAB, esto corresponde a la opción linear en la variable de salida del sistema Sugeno.

Ventaja del orden cero

Es más simple de diseñar, más fácil de entender y muy útil en ejemplos introductorios, clasificación simple y sistemas donde cada regla representa una acción bien definida.

Ventaja del primer orden

Puede describir mejor variaciones del sistema y lograr transiciones más suaves, porque la salida cambia en función de las entradas.

¿Cuál usar?

Si buscas simplicidad, usa orden cero. Si quieres una representación más fina y adaptable, usa primer orden. La elección depende del proceso, la precisión requerida y la facilidad de implementación.

¿Cómo se desarrolla la inferencia Sugeno?

Aunque Sugeno y Mamdani comparten la idea general de reglas difusas, el desarrollo interno cambia en la etapa final. Este es el flujo más claro para explicarlo en clase o en un informe.

1

Entradas

El sistema recibe valores reales como error, temperatura, nivel, rapidez o cualquier variable del proceso.

2

Fusificación

Se evalúa el grado de pertenencia de cada entrada en sus conjuntos difusos, por ejemplo bajo, medio, alto.

3

Evaluación de reglas

Se calcula la activación de cada regla usando operadores como AND o OR, normalmente min, max o producto.

4

Salida de cada regla

Cada regla produce una constante o una función lineal. Aquí está la gran diferencia frente a Mamdani.

5

Ponderación final

Las salidas de las reglas se combinan con un promedio ponderado según la fuerza de activación de cada regla.

Promedio ponderado

Si varias reglas se activan al mismo tiempo, no se elige una sola. Cada una aporta según su nivel de activación. Entonces, si una regla tiene más peso, influye más en la salida final. Esa es la razón por la cual Sugeno suele dar resultados continuos y suaves.

¿Hay defusificación?

En la práctica, no se usa una defusificación clásica como centroide, MOM, SOM o LOM. La salida final se obtiene directamente del promedio ponderado de los consecuentes numéricos. Por eso se dice que en Sugeno esa etapa se reemplaza por un cálculo matemático.

Comparación entre Sugeno y Mamdani

Esta comparación es clave porque normalmente en clase te preguntan por qué elegir uno u otro.

Aspecto Mamdani Sugeno
Consecuente de la regla Es un conjunto difuso, por ejemplo: salida es alta. Es un valor numérico o función, por ejemplo: salida = 0.8 o salida = 2x + y + 1.
Salida final Se obtiene luego de agregar y defusificar. Se obtiene por promedio ponderado de las reglas activadas.
Defusificación Sí, normalmente centroide. No en el sentido clásico; se reemplaza por una ponderación matemática.
Interpretación lingüística Muy intuitiva y fácil de explicar verbalmente. Menos lingüística en la salida, pero más práctica para cálculo y modelado.
Costo computacional Generalmente mayor. Generalmente menor y más eficiente.
Uso típico Sistemas didácticos, control interpretativo, decisiones lingüísticas. Modelado, control adaptable, optimización y aplicaciones con salida numérica directa.
Idea para recordar: Mamdani es más fácil de explicar con palabras; Sugeno es más fácil de implementar cuando se necesita una salida numérica directa y un cálculo más eficiente.

Cómo se configura Sugeno en MATLAB Fuzzy Logic Designer

En MATLAB, la configuración de Sugeno tiene una lógica bastante directa. Si ya viste la ventana del Fuzzy Logic Designer, estos son los pasos y detalles importantes que debes saber para explicarlo bien.

1. Crear un nuevo FIS Sugeno

En el menú principal del diseñador difuso se elige File → New FIS → Sugeno. Ahí MATLAB crea un sistema con estructura Sugeno, que por defecto puede iniciar con una entrada y una salida.

En ese momento también cambian las características del sistema respecto a Mamdani, especialmente en el tratamiento de la salida.

2. Configurar entradas

Las variables de entrada se definen casi igual que en Mamdani: nombre, rango y funciones de pertenencia. Puedes usar triangulares, trapezoidales, gaussianas u otras, dependiendo de cómo quieras representar el comportamiento del proceso.

3. Configurar la salida

La diferencia importante aparece en la salida. En el Membership Function Editor, la variable de salida puede tener funciones del tipo constant o linear. Eso indica si estás trabajando con Sugeno de orden cero o de primer orden.

Si eliges constant, cada regla tendrá un valor fijo. Si eliges linear, el consecuente dependerá de las entradas.

4. Definir reglas

Luego se pasa al Rule Editor. Allí se escriben reglas del tipo: SI error es pequeño Y delta es negativo ENTONCES salida = constante o función. En la práctica, MATLAB maneja la salida de esas reglas de acuerdo con el tipo de membership function del output.

5. Evaluar el sistema

Después puedes usar las herramientas de análisis como el Rule Viewer o el Surface Viewer para observar cómo responde el sistema. Esto sirve mucho para revisar si las reglas están coherentes y si la salida responde como se esperaba.

6. Qué revisar antes de terminar

  • Que el rango de las entradas y salidas sea correcto.
  • Que las funciones de pertenencia cubran todo el universo de discurso.
  • Que las reglas no se contradigan demasiado.
  • Que el tipo de salida sea el adecuado: constant o linear.
  • Que la superficie del sistema tenga el comportamiento que buscas.
